### Custom Hooks - Building Reusable Logic in React

Custom hooks are a powerful feature in React that allow developers to extract and reuse stateful logic across multiple components. By encapsulating common functionalities into a hook, you can enhance maintainability and promote code reuse within your application.

### When to Use It
- **State Management**: When you need to share stateful logic, such as form handling or fetching data, across multiple components.
- **Side Effects**: When you want to encapsulate logic for side effects that can be reused, like local storage interactions or API calls.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom Hook for fetching data
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) throw new Error('Network response was not ok');
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };
    fetchData();
  }, [url]);

  return { data, loading, error };
}

// Usage in a component
function DataDisplay({ apiUrl }) {
  const { data, loading, error } = useFetch(apiUrl);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error.message}</p>;
  return <pre>{JSON.stringify(data, null, 2)}</pre>;
}
```

### Explanation
- **Encapsulation**: Custom hooks allow the encapsulation of complex logic or shared state, making it easy to manage and maintain.
- **Reusability**: They enable you to use the same logic in different components without duplicating code, which leads to a cleaner codebase.

By using custom hooks, you can streamline your React components and promote efficient logic sharing.
