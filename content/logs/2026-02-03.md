### Custom Hooks - Reusable Logic in React

Custom hooks in React enable developers to extract component logic into reusable functions. This promotes code reuse and enhances maintainability by allowing the encapsulation of stateful logic and side effects in a separate module, making it easier to test and utilize across different components.

### When to Use It
- **Stateful Logic Sharing**: When multiple components need to share the same stateful logic, using a custom hook helps to keep the code DRY (Don't Repeat Yourself).
- **Complex Logic Handling**: When managing complex logic or side effects, custom hooks can encapsulate this logic in a more readable and manageable way.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom Hook
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch(url);
      const result = await response.json();
      setData(result);
      setLoading(false);
    };
    fetchData();
  }, [url]);
  
  return { data, loading };
}

// Component using the custom hook
function DataFetchingComponent() {
  const { data, loading } = useFetch('https://api.example.com/items');

  if (loading) return <div>Loading...</div>;

  return (
    <ul>
      {data.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

### Explanation
- **State Management**: The custom hook `useFetch` manages both the data and loading states, providing a clear interface to any component that calls it.
- **Effect Handling**: The hook utilizes the `useEffect` hook to trigger data fetching, ensuring the side effect is handled outside of the component logic.

Custom hooks serve as powerful tools in React to improve code quality and enhance the developer experience by allowing the separation of concerns.
