### Controlled Components - Managing State in Form Inputs

Controlled components are a key concept in React that help manage form input data through state. By controlling the state of form elements, developers can create more predictable and easier-to-maintain forms.

### When to Use It
- **Single Input Forms**: When you have a simple form, such as a login form, where the input values need to be validated and managed.
- **Complex Forms**: For more complex forms where multiple inputs depend on each other, controlled components allow for more interactivity and validation logic.

### Example
```jsx
import React, { useState } from 'react';

function ControlledForm() {
  const [inputValue, setInputValue] = useState('');

  const handleChange = (event) => {
    setInputValue(event.target.value);
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    alert('Submitted: ' + inputValue);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Enter Text:
        <input type="text" value={inputValue} onChange={handleChange} />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
}

export default ControlledForm;
```

### Explanation
- **State Management**: Controlled components tie the form inputs directly to the React component's state, allowing easy management and access to input values.
- **Event Handling**: The `onChange` event is used to update the state whenever the user types in the input, ensuring the displayed value is always in sync with the internal state.

In conclusion, adopting controlled components can enhance form management in React applications, leading to better control and validation of user input.
