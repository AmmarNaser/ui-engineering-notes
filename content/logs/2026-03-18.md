### Custom Hooks - Simplifying State Logic in React

Custom Hooks in React are a powerful way to encapsulate reusable stateful logic in your components. By extracting stateful logic into a custom hook, you can keep your components clean and focused while promoting code reuse across your application.

### When to Use It
- **State Logic Reuse**: When you have stateful logic that needs to be shared across multiple components, creating a custom hook can help centralize that logic and reduce duplication.
- **Complex State Management**: If a component contains complex state logic or multiple state variables, consider moving that logic to a custom hook to simplify the component and improve readability.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom Hook for window width
function useWindowWidth() {
    const [width, setWidth] = useState(window.innerWidth);

    useEffect(() => {
        const handleResize = () => setWidth(window.innerWidth);
        window.addEventListener('resize', handleResize);
        return () => window.removeEventListener('resize', handleResize);
    }, []);

    return width;
}

// Component using the custom hook
function App() {
    const width = useWindowWidth();

    return (
        <div>
            <h1>Window width: {width}px</h1>
        </div>
    );
}

export default App;
```

### Explanation
- **State Management**: The `useWindowWidth` custom hook manages the state of the window's width and listens for resize events.
- **Separation of Concerns**: This allows the `App` component to remain focused on rendering, while the logic related to window width is handled separately, promoting cleaner code.

Custom hooks are an excellent way to simplify state management in your React applications, making components easier to understand and maintain.
