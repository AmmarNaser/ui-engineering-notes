### Custom Hooks - Reusing Logic in React

Custom hooks in React are a powerful way to encapsulate and reuse component logic across different components. By allowing developers to extract stateful logic into a function, custom hooks enhance code readability and separation of concerns.

### When to Use It
- **State Management**: Use custom hooks when you have shared stateful logic that multiple components need.
- **Side Effects**: Create a custom hook to handle common side effects, like fetching data or subscribing to events, across various components.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom Hook
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        const result = await response.json();
        setData(result);
      } catch (error) {
        console.error('Error fetching data:', error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading };
}

// Component using the custom hook
function DataDisplayComponent() {
  const { data, loading } = useFetch('https://api.example.com/data');

  if (loading) return <p>Loading...</p>;
  return <pre>{JSON.stringify(data, null, 2)}</pre>;
}
```

### Explanation
- **Encapsulation**: Custom hooks allow you to encapsulate complex logic in a reusable function, keeping components cleaner and more focused.
- **Parameterization**: Custom hooks can accept parameters like `url` in the example above, making them flexible for different use cases while maintaining the same underlying logic.

Custom hooks provide a neat way to share logic across your components, enhancing maintainability and organization within your React applications.
