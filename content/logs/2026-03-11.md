### Custom Hooks - Reusable Logic in React
Custom hooks allow developers to encapsulate and reuse stateful logic in React applications. They help streamline code organization, making components cleaner and easier to read.

### When to Use It
- **State Management**: When you have complex state logic that needs to be shared between multiple components.
- **API Calls**: When you want to abstract the logic of fetching data and handling results in a reusable way.

### Example
```javascript
// useFetch.js
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

export default useFetch;

// ExampleComponent.js
import React from 'react';
import useFetch from './useFetch';

const ExampleComponent = () => {
  const { data, loading, error } = useFetch('https://api.example.com/data');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <ul>
      {data.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
};

export default ExampleComponent;
```

### Explanation
- **Reusable Logic**: Custom hooks can encapsulate any logic you'd like to reuse across multiple components, keeping your component code clean and DRY (Don't Repeat Yourself).
- **Hooks Compliance**: Custom hooks must start with "use" and can call other hooks, maintaining the rules of React hooks.

Custom hooks are a powerful feature in React that promotes code reuse and organization, leading to more maintainable applications.
