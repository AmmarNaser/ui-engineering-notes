### Hooks - Managing State in Functional Components

In React, Hooks are a powerful feature that allows you to use state and other React features without writing a class. They enable functional components to have stateful logic, making them more versatile and easier to manage.

### When to Use It
- **State Management**: Use Hooks when you need to manage local component state without converting to a class component. 
- **Side Effects**: Use the `useEffect` Hook to handle side effects, like data fetching or subscriptions, within functional components.

### Example
```jsx
import React, { useState, useEffect } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
    // Cleanup function to run on component unmount
    return () => {
      console.log('Cleanup for count:', count);
    };
  }, [count]); // Only run this effect when count changes

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
};

export default Counter;
```

### Explanation
- **useState**: This Hook allows you to declare state variables in functional components and provides a way to update them.
- **useEffect**: This Hook lets you perform side effects in components, such as interacting with the DOM or fetching data, while also allowing for cleanup logic.

Using Hooks greatly simplifies state management in your components, making your code cleaner and more maintainable.
