### Controlled Components - Managing Form Inputs in React

Controlled components are a fundamental concept in React where form data is handled by the state of the component. In controlled components, the form element's value is controlled by React, meaning that the input value is derived from the React state, allowing more streamlined handling and validation of user inputs.

### When to Use It
- **Handling Forms**: Use controlled components to manage user input in forms, providing clear validation and state management.
- **Dynamic Updates**: Implement in scenarios where you need to update other parts of the UI based on input values, making the interface more interactive.

### Example
```jsx
import React, { useState } from 'react';

const ControlledForm = () => {
  const [name, setName] = useState('');

  const handleChange = (event) => {
    setName(event.target.value);
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Form submitted with Name: ${name}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" value={name} onChange={handleChange} />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
};

export default ControlledForm;
```

### Explanation
- **State Management**: The input value is tied to the component's state via the `value` attribute, ensuring React is the single source of truth for input data.
- **Event Handling**: The `onChange` event updates the state every time the input changes, allowing for real-time updates and validation.

Controlled components provide a powerful way to manage input and form states, ensuring your application remains predictable and easy to debug.
