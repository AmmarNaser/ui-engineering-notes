### Custom Hooks - Reusable Logic in React

Custom hooks are a powerful feature in React that allow developers to encapsulate reusable logic outside of components. By creating custom hooks, you can maintain cleaner code and improve the maintainability of your applications.

### When to Use It
- **Stateful Logic Sharing**: When you need to share stateful logic across different components without repeating code.
- **Complex Logic Separation**: When your component's logic gets too complex, extracting that logic into a custom hook can simplify the component structure.

### Example
```javascript
import { useState, useEffect } from 'react';

function useFetch(url) {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);

    useEffect(() => {
        const fetchData = async () => {
            try {
                const response = await fetch(url);
                if (!response.ok) throw new Error('Network response was not ok');
                const result = await response.json();
                setData(result);
            } catch (error) {
                setError(error);
            } finally {
                setLoading(false);
            }
        };
        fetchData();
    }, [url]);

    return { data, loading, error };
}

export default useFetch;
```

### Explanation
- **Logic Encapsulation**: Custom hooks allow you to encapsulate complex or reusable logic, which leads to cleaner and more organized components.
- **Reusability**: Using custom hooks promotes reusability of logic since you can easily use the same hook in multiple components without duplicating code.

In conclusion, custom hooks enhance the reusability and readability of React components, making your code more maintainable and structured.
