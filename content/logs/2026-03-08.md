### Controlled Components - Managing Form Inputs in React

Controlled components are an essential concept in React for managing form inputs effectively. By using controlled components, you can handle user input through state management, ensuring that form data is consistent and easily manipulable.

### When to Use It
- **Form Handling**: When you need to manage complex form data and keep track of user input in real-time.
- **Validation Logic**: If your application requires immediate feedback or validation of input fields, controlled components allow you to run checks on every input change.

### Example
```jsx
import React, { useState } from 'react';

const ControlledForm = () => {
  const [name, setName] = useState('');

  const handleChange = (event) => {
    setName(event.target.value);
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Submitted Name: ${name}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" value={name} onChange={handleChange} />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
};

export default ControlledForm;
```

### Explanation
- **Single Source of Truth**: The value of the input is stored in the component's state, making it the single source of truth. This approach ensures that React is in sync with the input value.
- **Event Handling**: The `onChange` event handler updates the state every time the user types in the input field, allowing for dynamic rendering and validation based on the current input.

Using controlled components can greatly enhance the user experience and maintainability of your forms in React applications.
