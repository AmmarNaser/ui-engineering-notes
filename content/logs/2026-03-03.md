### React Hooks - Managing State with useState

React Hooks are a powerful addition to functional components, allowing you to manage state and side effects without writing class components. The `useState` hook is one of the most commonly used hooks, enabling developers to add state to functional components easily.

### When to Use It
- **Managing Local State**: Use `useState` when you need to store and manage local state within a functional component, such as form inputs or toggle states.
- **Handling User Input**: When you want to react to user actions like clicks or changes in input values, `useState` helps to keep track of those changes in real-time.

### Example
Here’s a simple example of using the `useState` hook to manage a counter in a functional component:

```jsx
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
      <button onClick={() => setCount(count - 1)}>
        Decrease
      </button>
    </div>
  );
};

export default Counter;
```

### Explanation
- **State Initialization**: The `useState` hook takes the initial state as an argument (in this case, `0`) and returns an array containing the current state and a function to update it.
- **State Updates**: You can update the state by calling the updater function (`setCount`), passing it the new state value. This triggers a re-render of the component with the updated state.

Using `useState` allows for more straightforward state management in functional components, making your code cleaner and easier to maintain.
