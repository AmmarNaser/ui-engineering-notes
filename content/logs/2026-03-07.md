### State Management - Understanding React Hooks for State

State management is a critical aspect of building React applications. Effective state management allows your components to react to user input, maintain data consistency, and provide a smooth user experience.

### When to Use It
- **Managing Local Component State**: When you need to store and manage state that is specific to a single component, such as form inputs or toggled UI elements.
- **Sharing State Across Components**: When multiple components require access to the same state, implementing a shared state using React's Context API combined with hooks can simplify data flow.

### Example
```jsx
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
};

export default Counter;
```

### Explanation
- **useState Hook**: The `useState` hook is used to create state variables in functional components, allowing you to add local state to your components.
- **State Updates**: State updates provided by `useState` lead to re-renders, which ensures the UI reflects the current state values, maintaining synchronization between the UI and state data.

Effective state management with hooks simplifies component logic, making your React applications more organized and maintainable.
