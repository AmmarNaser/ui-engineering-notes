### Custom Hooks - Simplifying Logic Reuse in React

Custom Hooks are a powerful feature in React that allow developers to encapsulate and reuse stateful logic across different components. By creating a Custom Hook, you can abstract away complex code and make your components cleaner and more maintainable.

### When to Use It
- **Shared State Logic**: When you have stateful logic that needs to be used across multiple components, a Custom Hook can help eliminate redundancy.
- **Side Effects Management**: If you are managing side effects like API calls in more than one place, consider wrapping this logic in a Custom Hook for better organization.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom Hook for fetching data
function useFetch(url) {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) throw new Error('Network response was not ok');
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      }
    };
    
    fetchData();
  }, [url]);
  
  return { data, error };
}

// Usage in a component
function App() {
  const { data, error } = useFetch('https://api.example.com/data');

  if (error) return <div>Error: {error}</div>;
  if (!data) return <div>Loading...</div>;

  return (
    <div>
      <h1>Data:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
}
```

### Explanation
- **Reusable Logic**: Custom Hooks allow for logic to be reused across different components without repeating code.
- **Stateful Logic**: They can manage state and lifecycle methods, making it easier to handle complex functionality in a clean way. 

Utilizing Custom Hooks can significantly improve the maintainability and readability of your React applications by encapsulating related logic in a reusable manner.
