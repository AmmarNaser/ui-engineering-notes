# React Frontend Engineering Note: Custom Hooks

## Concept
Custom hooks are a powerful feature in React that allows you to extract and reuse stateful logic across multiple components. By creating a custom hook, you can encapsulate related logic within a reusable function that can be called in any functional component.

## When to Use It
Use custom hooks when:
- You find yourself repeating the same stateful logic in multiple components.
- You want to abstract complex logic or API calls from your component.
- You need to share functionality between components without relying on higher-order components or render props.

## Small Example
Hereâ€™s a simple example of a custom hook that manages the visibility state of a modal.

```javascript
// useModal.js
import { useState } from 'react';

const useModal = () => {
  const [isVisible, setIsVisible] = useState(false);
  
  const openModal = () => setIsVisible(true);
  const closeModal = () => setIsVisible(false);

  return {
    isVisible,
    openModal,
    closeModal,
  };
};

export default useModal;
```

Then you can use this hook in a component like this:

```javascript
// ModalComponent.js
import React from 'react';
import useModal from './useModal';

const ModalComponent = () => {
  const { isVisible, openModal, closeModal } = useModal();

  return (
    <div>
      <button onClick={openModal}>Open Modal</button>
      {isVisible && (
        <div className="modal">
          <h1>Modal Content</h1>
          <button onClick={closeModal}>Close Modal</button>
        </div>
      )}
    </div>
  );
};

export default ModalComponent;
```

In this example, the `useModal` hook provides a clean and reusable way to control modal visibility, allowing the `ModalComponent` to focus on UI logic.
