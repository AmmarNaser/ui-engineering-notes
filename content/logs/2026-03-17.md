### useEffect - Managing Side Effects in React

In React, handling side effects is crucial for managing state and lifecycle events effectively. The `useEffect` hook simplifies this process by allowing you to perform actions based on component lifecycle events without the complexities associated with classes.

### When to Use It
- **Fetching Data**: When you need to retrieve data from an API after a component mounts.
- **Updating DOM**: When you require direct DOM manipulations or need to subscribe to events after rendering.

### Example
```javascript
import React, { useState, useEffect } from 'react';

const DataFetcher = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch('https://api.example.com/data');
      const result = await response.json();
      setData(result);
    };
    
    fetchData();
  }, []); // Empty dependency array to run only on mount

  return (
    <div>
      <h1>Data from API</h1>
      {data ? <pre>{JSON.stringify(data, null, 2)}</pre> : <p>Loading...</p>}
    </div>
  );
};

export default DataFetcher;
```

### Explanation
- **Lifecycle Management**: The `useEffect` hook serves as a replacement for component lifecycle methods like `componentDidMount`, enabling side effects in functional components.
- **Dependency Array**: By providing a dependency array, you can control when the effect runs, ensuring that it only executes when specific values change.

Using `useEffect` helps keep your functional components clean and efficient, promoting a better architecture in your React applications.
