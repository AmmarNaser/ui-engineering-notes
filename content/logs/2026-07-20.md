### React Hooks - Managing State in Functional Components

React Hooks are a powerful feature that allows you to use state and other React features without writing a class. They enable a more functional approach to component design and are essential for modern React development, especially in functional components.

### When to Use It
- **Managing Local State**: Use Hooks when you need to manage and update local state within a functional component, enabling cleaner and more manageable code.
- **Side Effects**: Utilize Hooks like `useEffect` to handle side effects such as API calls or subscriptions, keeping your components responsive to state changes.

### Example
```javascript
import React, { useState, useEffect } from 'react';

const Counter = () => {
    const [count, setCount] = useState(0);

    useEffect(() => {
        document.title = `Count: ${count}`;
    }, [count]);

    return (
        <div>
            <p>You clicked {count} times</p>
            <button onClick={() => setCount(count + 1)}>Click me</button>
        </div>
    );
};

export default Counter;
```

### Explanation
- **State Initialization**: The `useState` Hook initializes the state (`count`) and provides a function (`setCount`) for updating it.
- **Side Effect Management**: The `useEffect` Hook updates the document title whenever the `count` changes, demonstrating how to manage side effects related to state updates.

By leveraging React Hooks, you can create cleaner, more maintainable components that separate concerns without the need for class-based components.
