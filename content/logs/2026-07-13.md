### useEffect - Managing Side Effects in Functional Components

The `useEffect` hook is a fundamental concept in React that allows developers to handle side effects in functional components. Side effects can include data fetching, subscriptions, or manually changing the DOM, and understanding how to manage these is vital for building interactive applications.

### When to Use It
- **Data Fetching**: When you need to fetch data from an API upon component mounting or based on updated dependencies.
- **Event Listeners**: When you want to set up an event listener for a global event (e.g., window resize) and ensure it is cleaned up when the component unmounts.

### Example
```jsx
import React, { useState, useEffect } from 'react';

const DataFetchingComponent = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch('https://api.example.com/data');
      const result = await response.json();
      setData(result);
      setLoading(false);
    };
    fetchData();
  }, []); // Empty array makes it run once on mount

  if (loading) {
    return <div>Loading...</div>;
  }

  return (
    <div>
      <h1>Fetched Data:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default DataFetchingComponent;
```

### Explanation
- **Dependency Array**: The second argument to `useEffect` (an empty array in this case) controls when the effect runs. An empty array runs the effect only on mount.
- **Cleanup Function**: You can return a function from the effect to clean up resources, such as unsubscribing from a service or removing event listeners.

In summary, mastering `useEffect` enables React developers to handle side effects efficiently and maintain cleaner, more functional components.
