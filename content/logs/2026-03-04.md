### useEffect - Managing Side Effects in React

The `useEffect` hook is a fundamental part of React that allows you to perform side effects in function components. Understanding how to effectively use this hook is crucial for managing operations like data fetching, subscriptions, or manually changing the DOM.

### When to Use It
- **Data Fetching**: When you need to fetch data from an API when the component mounts or when a specific state changes.
- **Event Handling**: When you want to set up a subscription or listener that requires cleanup when the component unmounts.

### Example
```javascript
import React, { useState, useEffect } from 'react';

const DataFetchingComponent = () => {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch('https://api.example.com/data');
      const result = await response.json();
      setData(result);
      setLoading(false);
    };

    fetchData();
  }, []); // Empty array ensures this runs once on mount

  if (loading) {
    return <div>Loading...</div>;
  }

  return (
    <ul>
      {data.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
};

export default DataFetchingComponent;
```

### Explanation
- **Side Effects Handling**: `useEffect` allows you to handle side effects in a declarative way, managing how and when they run based on dependencies.
- **Cleanup Function**: The `useEffect` hook can return a cleanup function to clean up resources (e.g., subscriptions) when the component unmounts or before the effect re-runs.

Understanding `useEffect` is essential for building interactive and dynamic React applications, allowing for efficient updates and resource management.
