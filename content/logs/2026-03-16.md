### Custom Hooks - Creating Reusable Logic in React

Custom hooks enable developers to extract and reuse stateful logic across components in React applications. By encapsulating complex logic into a straightforward API, custom hooks facilitate better organization and reusability, making it easier to maintain code.

### When to Use It
- **Logic Reuse**: When you have logic that is shared between multiple components, creating a custom hook can help decentralize that logic and avoid code duplication.
- **State Management**: If you need to manage local component state or side effects that require similar handling in different components, a custom hook can encapsulate that behavior efficiently.

### Example
```javascript
// useFetch.js
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch(url);
      const result = await response.json();
      setData(result);
      setLoading(false);
    };

    fetchData();
  }, [url]);

  return { data, loading };
}

// App.js
import React from 'react';
import useFetch from './useFetch';

function App() {
  const { data, loading } = useFetch('https://api.example.com/data');

  if (loading) return <div>Loading...</div>;

  return (
    <ul>
      {data.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

export default App;
```

### Explanation
- **Key Point**: Custom hooks can accept parameters and return any data types (like objects), allowing for maximum flexibility in how the hook is used.
- **Another Point**: They promote cleaner component code by separating logic from UI, enhancing readability and maintainability.

In summary, custom hooks are a powerful feature in React that streamline the development process and improve code organization.
