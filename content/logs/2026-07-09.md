### Custom Hooks - Reusable Logic in React
Custom hooks are a powerful feature in React that allows developers to extract reusable stateful logic from components into their own functions. This promotes cleaner code and enables you to share logic between components without compromising the purity of your components.

### When to Use It
- **Shared Logic**: When multiple components need to perform similar logic, such as fetching data or managing form state.
- **Complex State Management**: When a component's logic is becoming too complex, it can be beneficial to encapsulate that logic into a custom hook for improved readability and maintainability.

### Example
```javascript
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
}

export default useFetch;
```

### Explanation
- **Reusability**: Custom hooks encapsulate logic that can be reused across different components, minimizing duplication and improving consistency.
- **Separation of Concerns**: By splitting logic into hooks, components remain focused on rendering UI, while hooks manage state and side effects.

By utilizing custom hooks, you can significantly enhance your React application’s organization and reusability of logic.
