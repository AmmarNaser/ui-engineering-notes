### Custom React Hooks - Reusable State Logic

Custom React Hooks allow developers to create reusable logic for stateful functions in React applications. They encapsulate stateful behavior, making it easier to share and manage state across components.

### When to Use It
- **Shared Logic**: When you have stateful logic that needs to be reused across multiple components, a custom hook can simplify and centralize state management.
- **Cleaner Components**: If your component is becoming too complex or is handling too much state, consider extracting some of that logic into a custom hook to maintain readability.

### Example
```javascript
import { useState, useEffect } from 'react';

// Custom hook to fetch data
const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) throw new Error('Network response was not ok');
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      }
    };

    fetchData();
  }, [url]);

  return { data, error };
};

// Component using the custom hook
const DataDisplay = ({ url }) => {
  const { data, error } = useFetch(url);

  if (error) return <div>Error: {error.message}</div>;
  if (!data) return <div>Loading...</div>;

  return (
    <div>
      <h3>Data:</h3>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};
```

### Explanation
- **Encapsulation of Logic**: Custom hooks allow you to encapsulate complex stateful logic, enabling code reuse across components without repetitive code.
- **Dependency Management**: They utilize React's dependency array, ensuring that the logic runs only when necessary, leading to efficient updates and state management.
