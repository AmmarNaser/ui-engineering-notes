React Hooks - Enhancing Functional Components with State Management

React Hooks are a powerful feature introduced in React 16.8 that allow you to use state and other React features in functional components. They help streamline component logic and promote cleaner, more reusable code.

### When to Use It
- **Managing Component State**: Use hooks when you need to manage state in functional components, allowing for cleaner and more readable code compared to class-based components.
- **Side Effects**: Implement hooks when you need to handle side effects, such as fetching data or subscribing to events, making your code more organized.

### Example
```jsx
import React, { useState, useEffect } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]); // Runs after every count change

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
};

export default Counter;
```

### Explanation
- **State Management**: The `useState` hook allows you to declare state variables directly in functional components.
- **Handling Effects**: The `useEffect` hook is used to perform side effects in function components, running after render and whenever dependencies change, keeping the UI in sync with state changes.

React Hooks facilitate a more functional approach to building components, improving code organization and readability while reducing complexity in state management.
