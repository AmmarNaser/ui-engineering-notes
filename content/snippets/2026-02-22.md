This snippet demonstrates how to create a reusable custom hook for fetching data from an API in React. It provides a clean and efficient way to make API calls and manage loading and error states.

### `useFetch.js` - Custom Hook for Fetching Data

```javascript
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

export default useFetch;
```

### Usage Example

```javascript
import React from 'react';
import useFetch from './useFetch';

const App = () => {
  const { data, loading, error } = useFetch('https://jsonplaceholder.typicode.com/posts');

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <div>
      <h1>Posts</h1>
      <ul>
        {data.map(post => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  );
};

export default App;
```

### Explanation
- **Custom Hook**: The `useFetch` hook encapsulates the logic for fetching data, making it reusable across different components.
- **Loading and Error States**: It manages the loading and error states internally, providing a user-friendly way to indicate the status of the data fetching process.
- **Automatic Refetch**: The hook automatically refetches the data whenever the `url` changes, making it adaptable to dynamic API endpoints.

This custom hook makes data fetching straightforward and cleanly separates logic from UI components.
