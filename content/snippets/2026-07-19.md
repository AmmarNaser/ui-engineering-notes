This snippet creates a custom React hook that fetches data from an API and manages loading and error states, making it easy to reuse throughout your application.

### `useFetch.js` - Custom Data Fetching Hook

```javascript
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

export default useFetch;
```

### Usage Example

```javascript
import React from 'react';
import useFetch from './useFetch';

const App = () => {
  const { data, loading, error } = useFetch('https://api.example.com/data');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <h1>Data from API:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default App;
```

### Explanation
- **Custom Hook**: The `useFetch` hook encapsulates the logic for data fetching, making it reusable across multiple components.
- **Loading State**: It manages the loading state, providing a clear indication when the data is being fetched.
- **Error Handling**: The hook includes error handling that returns a user-friendly error message if the API call fails.
- **Dependency Management**: The `useEffect` hook is used for side effects, and it automatically re-fetches data if the `url` parameter changes. 

This custom hook simplifies data fetching in your React applications, making it easier to manage API calls.
