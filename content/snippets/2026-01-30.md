This snippet provides a custom React hook for fetching data from an API, handling loading and error states effectively.

### `useFetch.js` - Custom Hook for Data Fetching

```javascript
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

export default useFetch;
```

### Usage Example

```javascript
import React from 'react';
import useFetch from './useFetch';

const App = () => {
  const { data, loading, error } = useFetch('https://jsonplaceholder.typicode.com/posts');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <h1>Posts</h1>
      <ul>
        {data && data.map(post => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  );
};

export default App;
```

### Explanation
- **Data Fetching**: The hook fetches data from the provided URL when the component mounts and whenever the URL changes.
- **Loading State**: While data is being fetched, a loading state is managed, which can be displayed to the user.
- **Error Handling**: If the fetch fails, the error state is updated with the error message, allowing for graceful error presentation.

This custom hook simplifies data fetching in your React applications, making it reusable across different components.
