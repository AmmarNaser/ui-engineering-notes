Here's a practical React code snippet that demonstrates how to implement a custom hook for fetching data from an API. This snippet is production-ready and handles loading and error states effectively.

```jsx
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

// Example usage in a component
const ExampleComponent = () => {
  const { data, loading, error } = useFetch('https://api.example.com/data');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <h1>Data:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default ExampleComponent;
```

### Explanation:
- The `useFetch` hook takes a URL as an argument and handles fetching the data.
- It maintains three pieces of state: `data`, `loading`, and `error`.
- The `useEffect` hook is used to fetch data when the component mounts, and it cleans up on unmount.
- The example component `ExampleComponent` demonstrates how to use the `useFetch` hook, rendering appropriate loading and error messages as needed.
