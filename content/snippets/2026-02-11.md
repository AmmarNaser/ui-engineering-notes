This snippet demonstrates a reusable custom hook for fetching data in a React application. It abstracts the data-fetching logic, making it easy to reuse across various components.

### `useFetch.js` - Custom Data Fetching Hook

```javascript
import { useState, useEffect } from 'react';

const useFetch = (url) => {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);

    useEffect(() => {
        const fetchData = async () => {
            try {
                const response = await fetch(url);
                if (!response.ok) {
                    throw new Error('Network response was not ok');
                }
                const result = await response.json();
                setData(result);
            } catch (err) {
                setError(err);
            } finally {
                setLoading(false);
            }
        };

        fetchData();
    }, [url]);

    return { data, loading, error };
};

export default useFetch;
```

### Usage Example

```javascript
import React from 'react';
import useFetch from './useFetch';

const App = () => {
    const { data, loading, error } = useFetch('https://api.example.com/data');

    if (loading) return <div>Loading...</div>;
    if (error) return <div>Error: {error.message}</div>;

    return (
        <div>
            <h1>Data Fetched:</h1>
            <pre>{JSON.stringify(data, null, 2)}</pre>
        </div>
    );
};

export default App;
```

### Explanation
- **Custom Hook**: The `useFetch` hook encapsulates data fetching logic, making it reusable across different components.
- **Loading and Error States**: It manages loading and error states, providing a clear API for handling data fetching outcomes.
- **Dynamic URL**: The hook takes a URL as an argument, allowing it to fetch different data simply by changing the input.

This custom hook simplifies data fetching in your components and promotes better code organization.
