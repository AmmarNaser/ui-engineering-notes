This snippet demonstrates how to create a reusable custom button component in React that supports loading states and click handlers.

### `CustomButton.js` - Reusable Button Component

```javascript
import React from 'react';
import PropTypes from 'prop-types';

const CustomButton = ({ label, onClick, isLoading, disabled }) => {
  return (
    <button 
      onClick={!isLoading && !disabled ? onClick : null} 
      disabled={isLoading || disabled} 
      style={{ 
        opacity: isLoading ? 0.6 : 1, 
        cursor: isLoading || disabled ? 'not-allowed' : 'pointer' 
      }}
    >
      {isLoading ? 'Loading...' : label}
    </button>
  );
};

CustomButton.propTypes = {
  label: PropTypes.string.isRequired,
  onClick: PropTypes.func,
  isLoading: PropTypes.bool,
  disabled: PropTypes.bool,
};

CustomButton.defaultProps = {
  onClick: () => {},
  isLoading: false,
  disabled: false,
};

export default CustomButton;
```

### Usage Example

```javascript
import React, { useState } from 'react';
import CustomButton from './CustomButton';

const App = () => {
  const [loading, setLoading] = useState(false);

  const handleClick = () => {
    setLoading(true);
    setTimeout(() => {
      setLoading(false);
      alert('Button clicked!');
    }, 2000);
  };

  return (
    <div>
      <h1>Custom Button Example</h1>
      <CustomButton 
        label="Click Me" 
        onClick={handleClick} 
        isLoading={loading} 
      />
    </div>
  );
};

export default App;
```

### Explanation
- **Reusable Component**: The `CustomButton` can be easily reused in different parts of the application by passing different labels and handlers.
- **Loading State**: The component handles a loading state, showing a loading text while the button action is being processed.
- **Disabled State**: The button can be disabled, preventing any actions when it is either loading or explicitly set to be disabled.

This button component enhances user experience by visually communicating its state.
