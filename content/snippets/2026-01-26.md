This code snippet demonstrates a reusable React custom hook for fetching data from an API. It handles loading states and error management, making it useful for various data-fetching needs in a React application.

### `useFetch.js` - Custom Hook for Data Fetching

```javascript
import { useState, useEffect } from 'react';

const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error('Network response was not ok');
        }
        const result = await response.json();
        setData(result);
      } catch (error) {
        setError(error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

export default useFetch;
```

### Usage Example

```javascript
import React from 'react';
import useFetch from './useFetch';

const App = () => {
  const { data, loading, error } = useFetch('https://api.example.com/data');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      <h1>Data Fetched from API:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default App;
```

### Explanation
- **Data Fetching**: The custom hook `useFetch` takes a URL as an input and manages the fetching of data from that URL.
- **Loading State**: A loading state is provided, allowing the user to display a loading message while the data is being fetched.
- **Error Handling**: The hook captures errors that may occur during the fetch operation, enabling the parent component to display meaningful error messages.

This hook can easily be reused across different components that need to fetch data from different APIs, enhancing code reusability and organization.
