This snippet demonstrates how to create a reusable button component in React that can handle multiple styles and click events efficiently.

### `Button.js` - Reusable Button Component

```jsx
import React from 'react';
import PropTypes from 'prop-types';
import './Button.css'; // Import your CSS for button styles

const Button = ({ label, onClick, styleType }) => {
    return (
        <button className={`btn ${styleType}`} onClick={onClick}>
            {label}
        </button>
    );
};

Button.propTypes = {
    label: PropTypes.string.isRequired,
    onClick: PropTypes.func.isRequired,
    styleType: PropTypes.oneOf(['primary', 'secondary', 'danger']).isRequired,
};

Button.defaultProps = {
    styleType: 'primary',
};

export default Button;
```

### Usage Example

```jsx
import React from 'react';
import ReactDOM from 'react-dom';
import Button from './Button';

const App = () => {
    const handleClick = () => {
        alert('Button clicked!');
    };

    return (
        <div>
            <Button label="Primary Button" onClick={handleClick} styleType="primary" />
            <Button label="Secondary Button" onClick={handleClick} styleType="secondary" />
            <Button label="Danger Button" onClick={handleClick} styleType="danger" />
        </div>
    );
};

ReactDOM.render(<App />, document.getElementById('root'));
```

### Explanation
- **Reusable Component**: The `Button` component can be used multiple times with different labels and styles, making it versatile across your application.
- **Style Management**: The component accepts a `styleType` prop to easily apply different styles, which can be defined in a corresponding CSS file.
- **Prop Types Validation**: PropTypes are used to ensure that the component receives the correct types of props, enhancing code reliability and maintainability.

This button component serves as a foundation for creating a variety of buttons that can be styled and reused throughout your application.
